# Инструкция по публикации проекта

## Проблема с сертификатом/манифестом

Проблема с сертификатом при запуске на другом компьютере была исправлена. Теперь проект настроен без подписи манифестов, что позволяет запускать программу на любом компьютере без проблем с сертификатами.

## Что было исправлено

1. ✅ Отключена подпись манифестов (`SignManifests=false`)
2. ✅ Отключена генерация манифестов (`GenerateManifests=false`)
3. ✅ Удалены ссылки на временный сертификат
4. ✅ Обновлена версия сборки до 2.0.0.0

## Способы распространения программы

### Вариант 1: Прямое копирование (рекомендуется)

1. Соберите проект в режиме **Release**:
   - В Visual Studio: Build → Configuration Manager → Release
   - Build → Rebuild Solution

2. Скопируйте папку `bin\Release` на целевой компьютер (в ней должен быть `PdfSharp.dll`)

3. Запустите `WindowsFormsApp1.exe`

**Сборка на другом ПК из исходников:** в репозитории должна быть папка `WindowsFormsApp1\lib\PdfSharp.dll`. Если её нет — обновите проект из git; папка `packages` в git не хранится.

**Преимущества:**
- Нет проблем с сертификатами
- Простое распространение
- Не требует установки

### Вариант 2: ClickOnce публикация (без подписи)

1. В Visual Studio: Правый клик на проект → Publish

2. В настройках публикации:
   - **Sign the ClickOnce manifests**: ❌ **СНИМИТЕ ГАЛКУ**
   - **Sign the assembly**: ❌ **СНИМИТЕ ГАЛКУ** (если есть)
   - Выберите место публикации (локальная папка или сетевой путь)

3. Нажмите **Publish**

**Важно:** Убедитесь, что в настройках проекта отключена подпись манифестов (уже исправлено в проекте).

### Вариант 3: Создание установщика (MSI)

1. Добавьте проект Setup в решение (Visual Studio Installer)

2. Настройте установщик:
   - Добавьте выходные файлы проекта
   - Добавьте необходимые зависимости (.NET Framework 4.8)
   - Настройте ярлыки и папки

3. Соберите установщик

## Проверка настроек проекта

Убедитесь, что в файле `WindowsFormsApp1.csproj` установлены следующие настройки:

```xml
<PropertyGroup>
    <GenerateManifests>false</GenerateManifests>
</PropertyGroup>
<PropertyGroup>
    <SignManifests>false</SignManifests>
</PropertyGroup>
```

Эти настройки уже применены в проекте.

## Требования на целевом компьютере

- Windows 7 или выше
- .NET Framework 4.8 (устанавливается автоматически при необходимости)
- SQL Server (локальный или удалённый)
- База данных SchoolInventoryDB (создаётся через скрипты в папке Database)

## Настройка подключения к БД на целевом компьютере

**Важно:** в репозитории строка подключения по умолчанию — `Server=localhost`. Имя вашего экземпляра SQL (например `SAHAR\SQLSERVER`) в исходниках не хранится — его задают только на том ПК, где установлен SQL Server.

После копирования программы на другой компьютер:

1. Установите БД (см. `ИНСТРУКЦИЯ_ПО_УСТАНОВКЕ_БД.md` в корне репозитория).
2. Откройте файл `WindowsFormsApp1.exe.config` (в папке с программой, **не** в Visual Studio).
3. Измените `Server=` на имя SQL Server на этом ПК (примеры в `App.config.example`):
```xml
<connectionStrings>
    <add name="DefaultConnection" 
         connectionString="Server=localhost;Database=SchoolInventoryDB;Integrated Security=true;" />
</connectionStrings>
```
   Частые варианты: `localhost`, `.\SQLEXPRESS`, `ИМЯ_ПК\ИМЯ_ЭКЗЕМПЛЯР`.
4. Сохраните файл и запустите программу снова.

## Устранение проблем

**Ошибка "Не удалось проверить подпись" или "Ошибка манифеста":**
- ✅ Исправлено: подпись манифестов отключена
- Если ошибка повторяется, пересоберите проект в режиме Release

**Ошибка "Приложение не может быть запущено":**
- Проверьте, что установлен .NET Framework 4.8
- Проверьте, что все файлы скопированы (включая .exe.config)

**Ошибка подключения к БД:**
- Проверьте строку подключения в .exe.config
- Убедитесь, что SQL Server запущен и доступен
- Проверьте, что база данных SchoolInventoryDB существует

## Рекомендации

1. **Для внутреннего использования:** Используйте вариант 1 (прямое копирование)
2. **Для распространения:** Используйте вариант 2 (ClickOnce без подписи) или вариант 3 (MSI)
3. **Для продакшена:** Рассмотрите возможность создания собственного сертификата для подписи (опционально)
