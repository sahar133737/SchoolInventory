# База данных SchoolInventoryDB

## Описание

Эта папка содержит SQL-скрипты для создания и заполнения базы данных программы "Школьная инвентаризация".

## Файлы

- **CreateDatabase.sql** - создание структуры базы данных (таблицы, индексы, связи)
- **SeedData.sql** - заполнение минимальными тестовыми данными для работы
- **InstallDatabase.sql** - объединённый скрипт для SSMS (выполняет оба скрипта подряд)
- **Install-Database.ps1** - PowerShell скрипт для автоматической установки
- **README.md** - этот файл с инструкциями

## Инструкция по использованию

### Вариант 1: Выполнение через SQL Server Management Studio (SSMS)

1. Откройте SQL Server Management Studio
2. Подключитесь к вашему экземпляру SQL Server
3. Откройте файл `CreateDatabase.sql`
4. Выполните скрипт (F5 или кнопка "Execute")
5. Откройте файл `SeedData.sql`
6. Выполните скрипт (F5 или кнопка "Execute")

### Вариант 2: Выполнение через командную строку (sqlcmd)

```cmd
sqlcmd -S ИМЯ_СЕРВЕРА -i CreateDatabase.sql
sqlcmd -S ИМЯ_СЕРВЕРА -i SeedData.sql
```

Например:
```cmd
sqlcmd -S localhost -i CreateDatabase.sql
sqlcmd -S localhost -i SeedData.sql
```

### Вариант 3: Выполнение через PowerShell (автоматический скрипт)

```powershell
# Перейдите в папку Database
cd WindowsFormsApp1\Database

# Выполните скрипт установки
.\Install-Database.ps1 -ServerInstance "localhost"
```

Или для удалённого сервера:
```powershell
.\Install-Database.ps1 -ServerInstance "ИМЯ_СЕРВЕРА\ЭКЗЕМПЛЯР"
```

### Вариант 4: Выполнение через PowerShell (вручную)

```powershell
Invoke-Sqlcmd -ServerInstance "ИМЯ_СЕРВЕРА" -InputFile "CreateDatabase.sql"
Invoke-Sqlcmd -ServerInstance "ИМЯ_СЕРВЕРА" -InputFile "SeedData.sql"
```

## Что создаётся

### Структура базы данных:
- **Categories** - категории инвентаря
- **Classrooms** - кабинеты (местоположения)
- **ResponsiblePersons** - ответственные лица
- **Users** - пользователи системы
- **Inventory** - инвентарь

### Тестовые данные:
- **10 категорий** (Мебель, Компьютерная техника, Спортивный инвентарь и т.д.)
- **15 кабинетов** (101-305, различные кабинеты)
- **15 ответственных лиц** (Иванов, Петров, Сидорова и т.д.)
- **4 пользователя**:
  - admin / admin (Администратор)
  - user1 / 123456 (Пользователь)
  - user2 / 123456 (Пользователь)
  - manager / 123456 (Администратор)
- **25 записей инвентаря** (различные предметы с полными данными)

## Важные замечания

1. **Путь к файлам БД**: В скрипте CreateDatabase.sql указан путь по умолчанию. При необходимости измените пути к файлам данных и логов на вашем сервере.

2. **Права доступа**: Убедитесь, что у вашей учётной записи есть права на создание баз данных.

3. **Существующая БД**: Скрипты проверяют существование объектов перед созданием, поэтому их можно запускать многократно без ошибок.

4. **Тестовые данные**: Скрипт SeedData.sql добавляет данные только если таблицы пусты. Для повторного заполнения сначала очистите таблицы.

## Настройка подключения в программе

После создания базы данных настройте строку подключения в файле `App.config`:

```xml
<connectionStrings>
    <add name="DefaultConnection" 
         connectionString="Server=ИМЯ_СЕРВЕРА;Database=SchoolInventoryDB;Integrated Security=true;" />
</connectionStrings>
```

Или для SQL Authentication:
```xml
<connectionStrings>
    <add name="DefaultConnection" 
         connectionString="Server=ИМЯ_СЕРВЕРА;Database=SchoolInventoryDB;User Id=логин;Password=пароль;" />
</connectionStrings>
```

## Устранение проблем

**Ошибка "Не удалось создать базу данных":**
- Проверьте права доступа к SQL Server
- Убедитесь, что служба SQL Server запущена
- Проверьте, что указанный путь к файлам существует

**Ошибка "Таблица уже существует":**
- Это нормально, скрипт пропустит создание существующих объектов
- Если нужно пересоздать, сначала удалите базу данных

**Ошибка "Нарушение ограничения внешнего ключа":**
- Убедитесь, что сначала созданы все справочные таблицы (Categories, Classrooms, ResponsiblePersons)
- Проверьте порядок выполнения скриптов
