## ВВЕДЕНИЕ

Современные образовательные учреждения располагают значительным объёмом материальных ценностей: мебелью, компьютерной и учебной техникой, спортивным инвентарём, оргтехникой [1]. Эффективный учёт имущества является обязательным условием сохранности государственного и муниципального имущества, рационального использования бюджетных средств и подготовки отчётности для контролирующих органов [2].

Исследование охватывает полный цикл работы с материальными ценностями школы — от их поступления (закупка, дарение) и закрепления за кабинетами и материально ответственными лицами (МОЛ) до внутреннего перемещения, списания (вследствие износа, поломки, утери) и проведения плановых и внеплановых инвентаризаций [3].

Эффективный и систематизированный учёт имущества критически важен для обеспечения учебного процесса, сохранности имущества и соблюдения требований законодательства в сфере бюджетного учёта [4]. Ведение учёта в бумажных журналах и разрозненных электронных таблицах приводит к трудностям в отслеживании местоположения дорогостоящего оборудования (интерактивных досок, проекторов, компьютеров), ошибкам при инвентаризации, потерям времени на составление отчётов и, как следствие, к возможным финансовым санкциям и неэффективному использованию бюджетных средств [5].

Основная деятельность, рассматриваемая в системе, — оперативный учёт материальных ценностей, фиксация их перемещения между кабинетами и МОЛ, оформление распоряжений о списании и формирование отчётной и инвентаризационной документации [6].

Актуальность темы обусловлена необходимостью цифровизации административно-хозяйственных процессов в школах и сокращения ручного труда при подготовке инвентаризационных ведомостей [7].

Целью прохождения практики является приобретение практических навыков в области проектирования и разработки информационной системы для автоматизации учёта материальных ценностей и инвентаризации в МБОУ «СОШ № 11 им. П.М. Камозина».

Для достижения поставленной цели необходимо решить следующие задачи:

- проанализировать существующие бизнес-процессы, связанные с поступлением, перемещением, списанием имущества и проведением инвентаризации;
- разработать архитектуру базы данных для хранения информации о материальных ценностях, кабинетах, материально ответственных лицах, фактах перемещения и списания;
- реализовать пользовательский интерфейс для эффективного взаимодействия сотрудников школы с системой;
- провести тестирование функционала системы и внести необходимые коррективы [8].

Объектом исследования является процесс учёта и инвентаризации материальных ценностей в МБОУ «СОШ № 11 им. П.М. Камозина».

Предметом исследования являются методы и средства автоматизации учёта школьного имущества с использованием настольного приложения и СУБД MS SQL Server [9].

Основные требования, предъявляемые к разрабатываемому программному продукту «Школьная инвентаризация» (проект VALERA):

- авторизация в системе по логину и паролю;
- разграничение прав доступа по ролям (администратор, пользователь) с настройкой матрицы прав;
- хранение и обработка данных инвентаря в базе данных;
- поиск, фильтрация и сортировка записей;
- добавление, изменение и удаление карточек инвентаря;
- оформление распоряжений имуществом (выдача, перемещение, назначение МОЛ, списание);
- формирование отчётов за выбранный период и экспорт в Excel и PDF;
- модуль статистики;
- ведение системного журнала;
- наличие справки о программе;
- обработка ошибок и валидация вводимых данных [10].

Для разработки используется среда Microsoft Visual Studio 2022 и язык программирования C# (платформа .NET Framework 4.8). Клиентская часть реализована в технологии Windows Forms. Для хранения данных применяется СУБД MS SQL Server [11].

Используемый стек технологий:

- архитектура: монолитная клиент-серверная модель;
- клиентская часть: Microsoft Windows Forms;
- сервер данных: MS SQL Server;
- отчётность: формирование в приложении, экспорт CSV (Excel), PDF [12].

Информационной базой исследования послужили нормативные документы по учёту имущества бюджетных учреждений, материалы по месту прохождения практики, а также научные и учебные публикации по автоматизации административных процессов [13].

Пояснительная записка состоит из двух глав. Первая глава посвящена теоретическому обоснованию разрабатываемого программного продукта: проводится анализ предметной области, обзор программ-аналогов, моделирование проектируемой системы и разработка технического задания. Вторая глава описывает разработку программного продукта: архитектуру, структуру данных, интерфейс, программный код и тестирование системы [14].

## 1 Теоретическое обоснование разрабатываемого программного продукта

### 1.1 Анализ предметной области

Инвентаризация в образовательном учреждении представляет собой процесс учёта, контроля и проверки наличия материальных ценностей, находящихся на балансе организации [15]. Данный процесс является важной частью управленческой деятельности, поскольку позволяет администрации контролировать состояние имущества, своевременно выявлять недостачи и излишки и обеспечивать эффективное использование ресурсов [16].

В качестве объекта исследования рассматривается МБОУ «СОШ № 11 им. П.М. Камозина» — общеобразовательная организация, в которой проходила производственная практика. В процессе деятельности школы используется значительное количество материальных ресурсов (оборудование, мебель, учебные пособия), что требует организации эффективной системы инвентаризации [17].

На текущий момент в школе используются следующие документы:

- инвентаризационные ведомости — заполняются ответственными лицами при проведении инвентаризации;
- отчёт по результатам инвентаризации — формируется для руководства учреждения [18].

Ведение учёта имущества является важной задачей для администрации, поскольку позволяет:

- контролировать наличие и состояние материальных ценностей;
- выявлять недостачи и излишки;
- отслеживать перемещение имущества между подразделениями (кабинетами);
- обеспечивать сохранность оборудования и материальных ресурсов;
- повышать эффективность использования имущества [19].

Обычно инвентаризация проводится вручную: сотрудники заполняют бумажные ведомости, после чего данные обрабатываются для формирования отчётности. Такой подход имеет ряд недостатков:

- значительные временные затраты на заполнение и обработку документов;
- высокая вероятность ошибок при внесении и обработке данных;
- ограниченные возможности анализа и оперативного получения информации [20].

В связи с этим актуальной задачей становится автоматизация процесса инвентаризации.

До внедрения программного продукта основные бизнес-процессы инвентаризации:

1. Фиксация наличия имущества:

- ответственные лица проводят проверку наличия материальных ценностей;
- результаты фиксируются в бумажных инвентаризационных ведомостях.

2. Формирование инвентаризационных документов:

- данные о наличии имущества заносятся в сводные ведомости;
- расхождения (недостачи и излишки) фиксируются вручную.

3. Обработка данных:

- документы передаются в бухгалтерию или администрацию;
- осуществляется обработка данных и формирование итоговых отчётов.

4. Контроль и принятие решений:

- руководство анализирует результаты инвентаризации;
- принимаются меры по устранению выявленных нарушений;
- информация о движении имущества фиксируется вручную.

Внедрение разрабатываемого программного продукта позволит устранить существующие недостатки, обеспечить соответствие нормативным требованиям и повысить эффективность работы сотрудников [3].

### 1.4 Разработка технического задания

#### Основания для разработки

Разработка программного продукта ведётся на основании задания, выданного на дипломное проектирование. Приказ от 25.03.2025 № 375-3 «Об утверждении тем выпускных квалификационных работ и руководителей обучающихся ПК БГТУ».

Организация, утвердившая тему: ГБПОУ «Брянский техникум профессиональных технологий и сферы услуг».

Наименование работы: разработка информационной системы «Учёт инвентаризации имущества» (на примере МБОУ «СОШ № 11 им. П.М. Камозина»).

#### Назначение разработки

Информационная система «Школьная инвентаризация» (проект VALERA) предназначена для автоматизации процессов учёта, контроля и мониторинга движения материальных ценностей (мебель, компьютерная техника, учебное оборудование и др.) в образовательном учреждении [10].

Пользователями системы являются:

- администратор — настройка системы, ведение справочников, управление пользователями и правами доступа;
- заведующий хозяйственной частью (материально ответственное лицо) — внесение данных о поступлении, перемещении и списании имущества;
- сотрудник бухгалтерии — формирование отчётов и контроль сведений;
- преподаватели и сотрудники — просмотр имущества, закреплённого за кабинетом или МОЛ (в пределах прав доступа).

#### Требования к программному изделию

**Требования к функциональным характеристикам**

Информационная система должна обеспечивать выполнение следующих функций:

- регистрация данных о материальных ценностях (инвентарный номер, наименование, стоимость, дата ввода в эксплуатацию, состояние);
- ведение справочников категорий, кабинетов, ответственных лиц;
- учёт перемещения имущества между кабинетами и МОЛ;
- учёт списания и выбытия имущества;
- поиск и фильтрация по наименованию, категории, кабинету, состоянию, периоду, стоимости;
- авторизация пользователей и разграничение прав доступа;
- формирование отчётов: сводный финансовый по категориям, по кабинетам и МОЛ, матрица «состояние × категория», детальный реестр с группировкой;
- модуль статистики за период;
- экспорт отчётов в Excel (CSV) и PDF;
- встроенная справка (клавиша F1);
- заполнение базы демонстрационными данными (для администратора) [10].

**Входные и выходные данные**

Входными данными являются:

- данные о материальных ценностях (инвентарный номер, наименование, описание, стоимость, дата принятия к учёту);
- данные о категориях имущества;
- данные о кабинетах (номер, наименование);
- данные о материально ответственных лицах (ФИО, должность);
- данные о перемещении и списании (вид операции, основание, номер распоряжения, дата);
- учётные данные пользователей (логин, пароль, роль);
- критерии поиска и фильтрации [11].

Выходными данными являются:

- инвентаризационная / сличительная ведомость (в виде отчёта в системе и файла экспорта);
- акт списания (оформление операции в модуле распоряжения);
- отчёты по категориям, кабинетам, состоянию за период;
- результаты поиска и фильтрации в таблицах;
- файлы экспорта Excel и PDF;
- записи системного журнала [12].

**Требования к временным характеристикам**

- программный продукт должен быстро реагировать на действия пользователя;
- задержка при загрузке списка инвентаря не должна превышать 10 секунд при объёме до 10 000 записей на локальном сервере;
- формирование отчёта не должно превышать 10 секунд;
- сохранение записи о поступлении или перемещении — не более 5 секунд;
- исключены зависания интерфейса при типовых операциях [11].

**Требования к пользовательскому интерфейсу**

- графический интерфейс Windows Forms в едином светлом стиле;
- форма авторизации с полями логина и пароля;
- главная форма с вкладками: Инвентарь, Распоряжение, Отчёты, Статистика; для администратора — дополнительные вкладки: Пользователи, Категории, Кабинеты, Ответственные, Журнал, Права доступа;
- диалоги добавления и редактирования инвентаря;
- панель фильтров и поиска на вкладке «Инвентарь»;
- левая панель управления на вкладке «Отчёты» (список отчётов, кнопки «Сформировать», «Экспорт Excel», «Экспорт PDF»);
- отображение сообщений об ошибках во всплывающих окнах;
- встроенная справка по клавише F1;
- адаптивное отображение при изменении размера окна [10].

**Требования к надёжности**

- аутентификация пользователей и разграничение прав по ролям;
- валидация вводимых данных (обязательные поля, формат инвентарного номера, диапазон цены);
- обработка ошибок подключения к БД и некорректных действий пользователя;
- журналирование операций в файлы и просмотр журнала в системе;
- рекомендуется резервное копирование базы данных SchoolInventoryDB средствами SQL Server [11].

**Условия эксплуатации**

Для эксплуатации необходим один оператор (завхоз, бухгалтер или администратор), владеющий ПК на уровне пользователя. Программа должна работать в ОС Windows 10/11. Допускается работа в локальной сети при установке SQL Server на сервере [12].

**Требования к составу и параметрам технических средств**

Минимальные системные требования:

- процессор: 2 ядра, 2,0 ГГц;
- оперативная память: 4 ГБ;
- свободное место на диске: 10 ГБ;
- операционная система: Windows 10/11;
- СУБД: MS SQL Server (Express, LocalDB или полная версия) [11].

**Требования к информационной и программной совместимости**

- сервер: MS SQL Server;
- клиент: .NET Framework 4.8, Windows Forms;
- экспорт отчётов: CSV (открытие в Microsoft Excel), PDF;
- кодировка UTF-8 с BOM для корректного отображения кириллицы в Excel [12].

**Требования к программной документации**

В состав документации входят:

- пояснительная записка;
- руководство пользователя (приложение Б);
- руководство программиста (приложение А);
- техническое задание (раздел 1.4 настоящей записки) [14].

**Этапы разработки программного продукта**

- изучение предметной области;
- обзор программ-аналогов;
- разработка технического задания;
- проектирование архитектуры и структуры БД;
- разработка пользовательского интерфейса и программного кода;
- тестирование и отладка;
- разработка программной документации [8].
